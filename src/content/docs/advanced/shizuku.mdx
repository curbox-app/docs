Shizuku is a FOSS Android tool that grants apps elevated permissions without root, using Android's ADB (Android Debug Bridge) protocol. When Curbox is integrated with Shizuku, it can enforce app blocking at the system level — making it significantly more robust than Accessibility Service-based blocking, and without requiring full root access.

This is the **recommended configuration** for power users who take their focus seriously.

---

## Why Shizuku?

| Method | Blocking strength | Privacy | Root required |
|--------|------------------|---------|---------------|
| Accessibility Service | Medium — can be disabled in Settings | Good — stays on device | No |
| **Shizuku (recommended)** | **Strong — system-level enforcement** | **Excellent** | **No** |
| Root (Magisk/LSPosed) | Very strong | Excellent | Yes |

With Shizuku, Curbox uses `ActivityManager` and `PackageManager` APIs directly, preventing blocked apps from launching at the Android system level.

---

## Prerequisites

- Android **11 or newer** for wireless ADB (no computer needed)
- Android **8–10** requires a one-time ADB setup via a computer (see Method B below)
- [Shizuku app](https://shizuku.rikka.app/) installed

---

## Step 1: Install and Start Shizuku

<Steps>

1. **Install Shizuku** from [F-Droid](https://f-droid.org/en/packages/moe.shizuku.privileged.api/) or [Google Play](https://play.google.com/store/apps/details?id=moe.shizuku.privileged.api).

2. **Open Shizuku** and choose your activation method:

</Steps>

### Method A: Wireless ADB (Android 11+, no computer needed)

<Steps>

1. Enable **Developer Options**: go to **Settings → About phone** and tap *Build number* seven times.

2. Enable **Wireless debugging**: **Settings → Developer options → Wireless debugging** → toggle on.

3. In the Shizuku app, tap **"Start via Wireless Debugging"** and follow the on-screen pairing steps. Shizuku will pair and activate automatically.

   <Aside type="tip">
   Wireless ADB is disabled on reboot. Shizuku will remind you to re-activate after each restart. This is an Android security constraint, not a Shizuku limitation.
   </Aside>

</Steps>

### Method B: USB ADB (Android 8–10, computer required — one time)

<Steps>

1. On your computer, install [Android Platform Tools](https://developer.android.com/tools/releases/platform-tools).

2. Enable **USB debugging** on your device: **Settings → Developer options → USB debugging**.

3. Connect your phone via USB. Run the following on your computer:

   <Code
     code={`adb shell sh /sdcard/Android/data/moe.shizuku.privileged.api/start.sh`}
     lang="bash"
     title="Start Shizuku via ADB"
   />

4. Confirm the pairing prompt on your device.

</Steps>

---

## Step 2: Grant Curbox Shizuku Permission

<Steps>

1. Open **Curbox → Settings → Blocking method**.

2. Select **"Use Shizuku"**. Curbox will request Shizuku authorization.

3. In the Shizuku dialog that appears, tap **"Allow"**.

4. Curbox will confirm with a green "Shizuku active" status badge.

</Steps>

<Aside type="caution">
Shizuku deactivates on reboot. Curbox will detect this and fall back gracefully to Accessibility Service mode, notifying you so you can re-enable Shizuku when convenient.
</Aside>

---

## Verifying the Integration

To confirm Shizuku is working correctly, run a test block:

<Steps>

1. Go to **Curbox → Developer & Testing** (hidden in Settings: tap the version number 5 times).
2. Tap **"Run Shizuku integration test"**.
3. Curbox will attempt to temporarily block and unblock a test app, then report the result.

</Steps>

A successful test will show:

```
✓ Shizuku API accessible
✓ ActivityManager binding established
✓ Test block applied successfully
✓ Test block removed successfully
```

## Troubleshooting

**"Shizuku not running"** after a restart
: Wireless ADB is reset on reboot. Re-activate via **Settings → Developer options → Wireless debugging** and restart Shizuku.

**Permission denied errors in Curbox**
: Revoke and re-grant the Shizuku permission: **Shizuku app → Authorized apps → Curbox → Revoke**, then re-grant from Curbox settings.

**Shizuku crashes on Android 13+**
: Ensure you are using a recent version of Shizuku. Older versions (before v13.3.5) have compatibility issues with Android 13's stricter ADB policies. Check [Shizuku's releases page](https://github.com/RikkaApps/Shizuku/releases) for the latest version.
