## How App blocker works

Curbox tracks what app you're using in real time and decides if the app needs to be blocked or allows according the configurations you set.


1. ** Create a new app blocker group **
  To create a new app group of blocked apps, Select Reducers and you'll be presented with a list of options, seledct app pause and then press the add button on bottom right.

2. ** Select Apps **
  You'll be presented with a varied range of options, all the options you select here would be applied to the entire range of apps selected.

3. ** Blocking Configuration **
  Curbox provides two app blocking type, prompty the usage based and time based
  
  The usage based app blocking blocks apps on the basis of how long you use an app. For example "Block instagram if i use it more than 1 hour a day"
  The time based app blocking allows apps to be used only between certain time intervals during a day. Like "Let me use instagram between 7pm to 9pm on Sunday and Saturday"

4. ** Press the configure settings button ** to set the limits (if you chose usage based blocking) or time ranges (if you selected time based blocking) for each individual day.


## The warning screen
In curbox the screen that appears when you press a blocked app anytime when outside the set time limit or app usage limit.
For example if you've set a limit of 1 hour using reddit, exceed it. You'll be shown the warning screen.

This is the screen that decides whether you should be allowed to pass or not.

The warning screen configuration might look scary at first but believe me its very easy yet extremely powerful
You can setup here what message(called the warning message) you'd like to appear when you press an app from that group.

Then you can specify on what basis you'll be allowed to unlock the app by choosing the unlock behaviour option.

Curbox provides a number of unlock behaviours.

Behaviours that completely block the app 
- Skip warning screen: This method will replicate what the focus mode does, simply press back. don't show anything or let user proceed. This wont show the custom warning message you set. 
- Disable Unlcoking entirely: This method will show you the warning message you've set but not let you proceed

Behaviours that allow to extend app usage once the set configuration has been exceeded. 
- User Selects unlock time: This would show a time picker on the warning screen that lets you specify how long more you want to use the app. 
- Fixed unlock time: You preselect how long the time extend would last.

- Qr/Bar code: You can setup a system where scanning different qr/bar codes let you use extend the app usage for different time. For example i can set it up let me use the app for 10 mins if i scan the code on my sunscreen, or 20 mins if i scan the code on a pack of flour. You can generate and print new qr codes too.
- Unlock requires typing a sentence: This forces you to repeat a specific sentence to extend the app usage. eg, requiring to type: "I'm a loser for giving up my promise to lock in" to extend the app for x hours.
- Unlock requires stating intent: This mechanism asks user why they need to extend the app usage before letting them in. This written intent would be further on logged onto reducers>logged intents. The intent you type would be displayed as an overlay over your app the entire time you use it during the selected time reminding you your original intention.


If you are someone that impulsively decides to extend an app usage, the wait before unlocking feature basically FORCES YOU TO WAIT for set time before letting you in. This prompts the user to read the intented warning message.
The vibrate and max brightness on warning option basically wakes you up from the zombie state you get into when scrolling addicitve apps to let you make a proper decision about extending app usage.
