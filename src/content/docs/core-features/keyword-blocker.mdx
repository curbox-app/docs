The Keyword Blocker watches the text on your screen and in your browser's address bar. The moment it spots a word you have banned, it immediately redirects you — away from the distraction and towards a URL of your choosing.


## What It Blocks

The Keyword Blocker operates in two places simultaneously:

**In supported browsers**, it monitors the address bar and the page title. If a URL or title contains a blocked keyword, Curbox automatically types your redirect URL into the address bar and navigates you away.

**In any app** (when deep search is enabled), it scans all visible text fields on the screen. If blocked content appears anywhere, Curbox sends you home.


## Supported Browsers

The following browsers are supported for URL and title-based interception:

- Google Chrome
- Brave
- Vanadium (GrapheneOS)
- Opera

<Aside type="note">
Firefox support is partially implemented. URL detection works, but the automatic redirect may not complete in Firefox due to restrictions on editing the address bar. Curbox will fall back to sending you home instead.
</Aside>

If you use a browser not in the list, you can enable **Block All Except Supported Browsers**. This tells Curbox to send you home whenever you open any unrecognised browser, preventing you from side-stepping the blocker by switching apps.


## Setting Up the Keyword Blocker

<Steps>
1. **Navigate to Reducers**
   Open Curbox and tap **Reducers** in the bottom navigation bar.

2. **Open Keyword Blocker**
   Find the **Keyword Blocker** card and tap it.

3. **Add your keywords**
   Tap **Add Keyword** and type the word you want to block. Repeat for each word. Keywords are matched case-insensitively, so "Reddit", "reddit", and "REDDIT" all behave the same way.

4. **Set your redirect URL**
   Enter the URL you want to be taken to when a keyword is found. The default is `https://curbox.life`. You might prefer something genuinely useful, like your note-taking app's web interface, a productivity dashboard, or simply `https://google.com`.

5. **Configure extra options** (optional)
   - **Search all text fields**: scan every piece of visible text in every app, not just browsers. Useful if you want to catch keywords appearing in social media feeds or search results.
   - **Block unsupported browsers**: send you home if you open a browser Curbox does not have a built-in rule for.
   - **Ignored apps**: list any apps whose text should never be scanned.

6. **Enable and save**
   Toggle the main switch and tap **Save**.
</Steps>


## How Keywords Are Matched

When the blocker scans a URL it is smarter than a simple substring match. For a URL like `https://www.reddit.com/r/gaming/`, Curbox extracts:

- The full domain: `www.reddit.com`
- The site name: `reddit`
- Each path segment: `r`, `gaming`
- Query parameter names and values

So adding the keyword `reddit` will catch `reddit.com`, `old.reddit.com`, and any subdomain of Reddit.

For plain text (when deep search is enabled), Curbox splits the visible text into individual words and checks each one against your keyword list.

<Aside type="tip">
A small keyword list of your biggest time-wasters (for example `reddit`, `twitter`, `youtube`) is usually more effective than a long list. Fewer, high-impact rules are easier to maintain and less likely to produce false positives.
</Aside>


## Performance

The blocker uses an internal cache of up to 200 recently evaluated strings. Strings that were already checked are answered instantly from the cache rather than re-evaluated, keeping the blocker responsive without draining your battery.
